# 小说知识引擎

知识图谱驱动的小说创作辅助平台。

## 这是什么

这是一个基于 **知识图谱 + 向量检索** 的小说创作辅助工具。它将你的素材和小说文本转化为可视化的概念网络，帮你理清人物关系、追踪伏笔、检测逻辑漏洞。

基于 [MiroFish-Refactor-zh](https://github.com/M-YiXi/MiroFish-Refactor-zh) 改造，保留了核心的图谱引擎和节点图界面，去掉了模拟/预测功能，加入针对小说场景的专属能力。

## 双入口设计

### 入口一：素材大纲

导入参考资料、神话传说、设定文档，自动提取实体并构建关联网络。适合写作前的世界观搭建和素材整理。

- 多源导入（PDF/TXT/MD）
- 实体提取（角色、地点、事件、物品）
- 自动关联推理
- 素材汇总报告

### 入口二：小说推演

导入整本小说，构建人物关系图谱，与角色对话，检测逻辑漏洞，发现未回收的伏笔。

- 角色对话（选定角色 + 向量记忆增强，严格基于原文信息，不编造）
- 智能问答（自然语言查询，向量检索 + LLM 总结）
- 逻辑漏洞检测（死亡角色后续活动 + 设定一致性）
- 伏笔追踪（低关联度节点 + LLM 判断是否回收）
- 角色一致性检测（同一角色描述矛盾比对）
- 相似情节检测（向量余弦相似度找重复场景）

## 技术栈

- **前端**：Vue 3 + D3.js（力导向节点图）
- **后端**：Python Flask
- **图数据库**：Neo4j 5.x（含向量索引）
- **LLM**：支持任意 OpenAI SDK 兼容的 LLM API（DeepSeek / 火方引擎 / 阿里百炼等）
- **向量嵌入**：支持独立配置（Ollama / API 均可）
- **流式进度**：SSE（Server-Sent Events）实时推送构建进度

## 适用场景

- 写长篇网文/小说前整理世界观
- 检查设定一致性和逻辑漏洞
- 分析已写章节的人物关系
- 从大量参考素材中发现隐藏关联
- 追踪伏笔是否回收
- 与小说角色对话，验证角色行为是否符合设定





## API 接口（核心接口）

| 方法 | 路径 | 功能 |
|:-----|:-----|:-----|
| POST | `/api/graph/ontology/generate` | 上传文件 + LLM 生成本体 |
| POST | `/api/graph/build` | 异步构建图谱 |
| GET | `/api/graph/build/stream/<task_id>` | SSE 流式推送构建进度 |
| GET | `/api/graph/task/<task_id>` | 查询任务状态 |
| GET | `/api/graph/data/<graph_id>` | 获取图谱节点/边数据 |
| GET | `/api/graph/characters/<graph_id>` | 获取角色列表（选角用） |
| POST | `/api/graph/character/chat` | 角色对话（向量记忆增强） |
| POST | `/api/graph/qa` | 智能问答（向量+LLM） |
| POST | `/api/graph/plot/check` | 逻辑漏洞 + 伏笔检测 |
| POST | `/api/graph/foreshadow/track` | 伏笔追踪 |
| POST | `/api/graph/character/consistency` | 角色一致性检测 |
| POST | `/api/graph/similarity/detect` | 相似情节检测 |
| POST | `/api/graph/relation/reason` | 两节点关联推理 |
| POST | `/api/graph/outline/summary` | 素材汇总报告 |
| POST | `/api/graph/reader/comments` | 多视角读者评论 |
| GET | `/api/graph/project/<id>` | 获取项目详情 |
| GET | `/api/graph/project/list` | 列出所有项目 |
| DELETE | `/api/graph/project/<id>` | 删除项目 |
| POST | `/api/graph/project/<id>/reset` | 重置项目状态 |
| GET | `/api/graph/tasks` | 列出所有任务 |
| DELETE | `/api/graph/delete/<graph_id>` | 删除图谱 |
